# 轮次控制快速开始指南

## 🚀 5分钟快速上手

### 第1步：了解核心概念

**启动一次 = 持续运行 = 自动循环**

- 启动程序
- 自动执行第1轮 → 等待8小时
- 自动执行第2轮 → 等待8小时
- 自动执行第3轮 → 等待到明天
- 新的一天，重置轮次，继续执行
- 循环往复，程序一直运行

### 第2步：测试功能（快速模式）

修改配置文件 `脚本配置/轮次配置.json`：

```json
{
  "启用轮次控制": true,
  "每日轮次": 3,
  "轮次间隔策略": "固定间隔",
  "轮次间隔小时": 0.05,  // 3分钟
  "打乱账号顺序": true
}
```

启动程序，观察日志：

```
执行第1轮 → 完成
⏳ 等待第 2 轮... 剩余: 3分钟
执行第2轮 → 完成
⏳ 等待第 3 轮... 剩余: 3分钟
执行第3轮 → 完成
🌙 等待新的一天... 剩余: X小时
```

### 第3步：恢复正常配置

测试完成后，改回正常配置：

```json
{
  "启用轮次控制": true,
  "每日轮次": 3,
  "轮次间隔策略": "平均分配",
  "打乱账号顺序": true
}
```

## ❓ 常见问题

### Q: 程序会一直运行吗？
A: 是的。程序会持续运行，自动执行3轮，然后等待到明天，循环往复。

### Q: 可以看到等待倒计时吗？
A: 可以。等待期间每分钟更新一次倒计时，显示剩余时间。

### Q: 软件崩溃后会重复执行吗？
A: 不会。程序会记住每个账号的执行进度，重启后自动跳过已完成的账号。

### Q: 可以改成每天2轮或4轮吗？
A: 可以。修改配置文件中的 `"每日轮次": 3` 为其他数字。

## 🎯 推荐配置

```json
{
  "轮次间隔策略": "平均分配",
  "每日轮次": 3
}
```
- 间隔：8小时
- 执行时间：自动分配

### 随机策略（最安全）
```json
{
  "轮次间隔策略": "随机间隔",
  "每日轮次": 3
}
```
- 间隔：6-10小时随机
- 更接近真实用户

## 🔧 故障排除

### 问题1：程序停止运行了
**原因**：可能被误关闭或崩溃  
**解决**：重新启动程序，会自动从断点继续

### 问题2：某些账号一直被跳过
**原因**：这些账号已完成当前轮次  
**解决**：正常现象，下一轮会执行

### 问题3：想重新执行某个账号
**解决**：编辑 `账号.json`，将该账号的 `今日执行轮次` 改为 0
